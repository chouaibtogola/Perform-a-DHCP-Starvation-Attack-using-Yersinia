# Network Level Attacks and Countermeasures

## Lab: Perform a DHCP Starvation Attack using Yersinia

---

## Objective

The objective of this lab is to perform a DHCP starvation attack using the Yersinia tool and analyze the generated network traffic using Wireshark.

---

## Background

The Dynamic Host Configuration Protocol (DHCP) automatically assigns IP addresses to devices on a network. DHCP servers maintain a pool of available IP addresses that are distributed to clients upon request.

A DHCP starvation attack occurs when an attacker floods the DHCP server with a large number of fake DHCP requests. As a result, the DHCP server exhausts its available IP address pool and becomes unable to assign addresses to legitimate clients, causing a Denial-of-Service (DoS) condition.

Yersinia is a network attack framework capable of exploiting weaknesses in several network protocols, including DHCP. It can be used to automate DHCP starvation attacks by continuously generating DHCP requests with spoofed MAC addresses.

## Results

The DHCP starvation attack successfully generated a large number of DHCP requests using spoofed MAC addresses. Wireshark captured the resulting traffic, showing repeated DHCP Discover and DHCP Request messages.

The attack demonstrates how an attacker can exhaust the DHCP server's available IP address pool, preventing legitimate clients from obtaining network configurations and causing a Denial-of-Service condition.

---

## Countermeasures

To mitigate DHCP starvation attacks, organizations should implement:

- DHCP Snooping
- Port Security
- Rate Limiting on Switch Ports
- Network Access Control (NAC)
- Dynamic ARP Inspection (DAI)
- Intrusion Detection and Prevention Systems (IDS/IPS)
- DHCP Lease Monitoring

---

## Conclusion

In this lab, a DHCP starvation attack was performed using the Yersinia framework. The attack generated numerous DHCP requests with spoofed MAC addresses, which were captured and analyzed using Wireshark. The exercise demonstrated how attackers can exhaust the DHCP address pool and deny network access to legitimate users. Implementing DHCP Snooping and other network security controls can significantly reduce the risk of this attack.
